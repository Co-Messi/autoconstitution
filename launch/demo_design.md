# autoconstitution Demo Design

## Demo title

**Autoresearch, but with multiple agents**

## Core message

Show one thing clearly:

`autoconstitution` takes a single-agent improvement loop and turns it into a role-based system where agents can challenge and teach each other under explicit rules.

---

## Demo structure

### Part 1 - The setup

Show:

- a prompt or task
- `constitution.md`
- the selected provider

Message:

"The rules are explicit, and the task is concrete."

### Part 2 - The loop

Show:

- Student output
- Critic feedback
- Teacher or alternative direction
- Judge decision

Message:

"This is not just another prompt wrapper. The system has internal pressure."

### Part 3 - The result

Show:

- chosen answer
- rejected answer
- critique trace
- optional exported preference pair

Message:

"The output is not just one answer — it is an improvement trace."

---

## Best first demo task

Pick one:

- improve a coding-agent prompt
- design a better financial-analysis workflow

The ideal first demo is:

- easy to understand
- clearly improved by critique
- not dependent on giant infrastructure

---

## What the audience should understand

By the end of the demo, they should be able to say:

- this is like autoresearch, but multi-agent
- the constitution controls the rules
- the loop can be reused for other domains

If they only think "interesting orchestration framework," the demo failed.
