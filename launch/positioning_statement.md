# autoconstitution Positioning Statement

**autoconstitution** is autoresearch with multiple agents. It turns the single-agent "try something, measure it, keep or revert it" loop into a small society of agents that can propose, critique, revise, judge, and preserve improvements under explicit rules in `constitution.md`.

At the product level, `autoconstitution` is a reusable improvement loop for prompts, agents, workflows, and domain-specific systems such as financial analysis, coding, and research assistants. At the implementation level, it combines constitutional critique, ratcheted evaluation, provider abstraction, and optional multi-agent orchestration so the same pattern works on local Ollama setups as well as stronger cloud-backed model stacks.

The core promise is simple: define the task, define the rules, let the agents challenge and teach each other, and keep only the improvements that survive judgment.
